# Reg Monkey 使用手册

这份手册说明如何使用 Reg Monkey 的公开演示版本或本地 Streamlit 版本。

Reg Monkey 帮你按结构化流程完成实证分析：准备数据、理解数据质量、选择模型路径、运行分析、阅读结果、导出报告。它不证明因果关系，也不替代研究设计判断。

## 1. 打开应用

打开托管演示链接，或在本地运行：

```bash
streamlit run app.py
```

如果开启了公开演示模式，应用会显示公开演示提示。建议先使用示例数据，不要上传敏感、机密或个人数据。

## 2. 在设置页选择数据

你有两种开始方式：

- **使用示例数据**：适合第一次走完整流程。示例数据是合成数据，只用于演示工作流。
- **上传自己的数据**：上传整理好的 CSV 或 Excel 文件。公开演示环境中请使用小型、非敏感文件。

加载数据后，先确认预处理结果，再确认变量角色。变量角色会告诉应用哪一列是结果变量，哪些列是解释变量，哪些列是控制变量或标识变量。

## 3. 理解数据

“理解数据”页面用于检查数据是否适合进入建模。

你可以查看：

- 行数和列数
- 缺失情况
- 变量摘要
- 数值变量和分类变量
- 相关关系
- 资源或数据规模提示

如果数据看起来不对，可以回到设置页，调整预处理方式，或上传更干净的文件。

## 4. 制定分析计划

“分析计划”页面有两条主要路径：

- **应用推荐方案并进入运行分析**：使用生成的主模型设定，并进入运行页面。
- **选择手动配置**：自己选择模型类型和变量。

DID、IV/2SLS 和 PSM 是带保护提示的手动研究设计流程。它们可以用于结构化研究设计，但 Reg Monkey 不会自动确认这些方法背后的假设。

## 5. 手动配置模型

手动配置可以选择：

- 模型类型
- 结果变量
- 核心解释变量或处理变量
- 控制变量
- 面板、时间、组别、工具变量或匹配变量等必要字段

如果必填字段缺失，运行页面会保持不可运行状态，或显示运行前风险提示。

## 6. 运行分析

运行前请检查：

- 当前模型类型
- 已包含的变量
- 样本量
- 运行前风险提示

点击 **运行模型** 执行分析。运行成功后，可以点击 **进入解释结果** 阅读输出。

## 7. 解释结果

“解释结果”页面先显示结果快照，然后显示初学者阅读提示、关键发现、研究解释、诊断和技术细节。

阅读结果时请注意：

- OLS 重点看系数方向、不确定性、R² 和诊断。
- Logit/Probit 的原始系数默认不是概率百分点变化。
- 面板固定效应使用个体内部变化。
- DID 需要研究设计支持，尤其是平行趋势判断。
- IV/2SLS 需要判断工具变量相关性和排除限制。
- PSM 只平衡可观测协变量，未观测混杂仍可能存在。

分享结论前，请先检查诊断、假设和谨慎提示。

## 8. 导出报告

“导出报告”页面提供：

- 简版报告下载
- 完整技术报告下载
- 可复现压缩包下载
- 结果表和清理后数据的可选 CSV 下载

分享前请先检查报告内容。公开演示导出的结果不应被视为因果证明，也不能替代人工审阅。

## 9. 切换语言

使用侧边栏的语言选择器，可以在中文和英文之间切换。切换语言时，当前工作流状态应继续保留。

## 10. 重新开始

如果想重新开始，可以点击 **重新开始**。确认步骤会说明哪些状态会被清除。重新开始会清除应用内分析流程状态，但不会删除你电脑上的本地文件。

## 11. 常见文件问题

如果文件无法读取，Reg Monkey 会在上传区域附近显示友好的错误提示。常见原因包括：

- 文件类型不支持
- 空文件
- CSV 或 Excel 文件格式异常
- 重复或空白列名
- 没有可用行或可用列
- 需要数值输入的模型找不到数值变量

修复文件后可以重新上传，也可以改用示例数据。

## 12. 演示安全边界

- 建议先使用示例数据。
- 不要在公开演示环境上传敏感、机密或个人数据。
- 托管演示存在资源限制。
- 统计估计不是自动因果结论。
- 分享前请检查诊断、假设和导出内容。
